## Maintainer Responsibilities

Maintainers are responsible for:

1. **Code Quality**
   - Review and approve pull requests
   - Ensure code meets project standards
   - Maintain test coverage
   - Keep dependencies updated

2. **Documentation**
   - Keep documentation accurate and up-to-date
   - Review documentation contributions
   - Ensure examples work correctly

3. **Community**
   - Respond to issues and questions
   - Foster a welcoming environment
   - Recognize and encourage contributors
   - Coordinate with Open Horizon community

4. **Releases**
   - Plan and execute releases
   - Maintain changelog
   - Tag versions appropriately
   - Coordinate with downstream users

5. **Security**
   - Address security vulnerabilities promptly
   - Coordinate security disclosures
   - Keep dependencies secure
   - Follow security best practices

## Decision Making

### Minor Changes
- Documentation updates
- Bug fixes
- Small feature additions
- Dependency updates

**Process**: Single maintainer approval required

### Major Changes
- Architecture changes
- Breaking changes
- New major features
- Policy changes

**Process**: Consensus among maintainers required

### Emergency Changes
- Critical security fixes
- Service outages
- Data loss prevention

**Process**: Any maintainer can act immediately, notify others ASAP

## Becoming a Maintainer

We welcome new maintainers who have demonstrated:

1. **Sustained Contribution**
   - Regular, high-quality contributions over 6+ months
   - Multiple merged pull requests
   - Active participation in discussions

2. **Technical Expertise**
   - Deep understanding of the codebase
   - Knowledge of Open Horizon platform
   - Understanding of edge computing concepts

3. **Community Engagement**
   - Helpful responses to issues
   - Constructive code reviews
   - Positive community interactions

4. **Alignment with Project Goals**
   - Understanding of project vision
   - Commitment to project values
   - Availability for maintainer duties

### Nomination Process

1. **Self-Nomination or Nomination by Current Maintainer**
   - Open an issue titled "Maintainer Nomination: [Name]"
   - Include contribution history and qualifications

2. **Discussion Period**
   - 2-week discussion period
   - Current maintainers evaluate nomination
   - Community feedback welcome

3. **Decision**
   - Consensus among current maintainers
   - Nominee notified of decision
   - If approved, added to this document

## Maintainer Emeritus

Maintainers who step down are recognized as Maintainer Emeritus:

- Listed in acknowledgments
- Retain commit access (optional)
- Welcome to return as active maintainer
- Continued recognition for contributions

### Process for Stepping Down

1. Notify other maintainers
2. Transfer active responsibilities
3. Update this document
4. Optional: Retain commit access
